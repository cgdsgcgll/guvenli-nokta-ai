# Sprint Retrospective

## İyi Gidenler

- Proje fikri kısa sürede netleştirildi.
- GitHub repository oluşturuldu.
- Product backlog hazırlandı.
- Sprint board oluşturuldu.
- İlk arayüz taslakları hazırlandı.
- Takım rolleri belirlendi.

## Geliştirilmesi Gerekenler

- Takım içi görev dağılımı daha net yapılmalı.
- Her ekip üyesinin düzenli commit atması sağlanmalı.
- Sprint 2'de kod geliştirme süreci hızlandırılmalı.
- Frontend, backend ve AI görevleri ayrı ayrı takip edilmeli.
- Daily Scrum notları daha düzenli tutulmalı.

## Sprint 2 İçin Aksiyonlar

- GitHub Projects üzerinde görevler ekip üyelerine atanacak.
- Frontend ekran tasarımlarına başlanacak.
- Backend API yapısı kurulacak.
- Risk analizi kriterleri kod tarafına aktarılacak.
- AI öneri sistemi için ilk prototip hazırlanacak.

## Genel Değerlendirme

Sprint 1, ürün fikrinin ve proje yönetimi yapısının oluşturulması açısından verimli geçmiştir. Sprint 2'de geliştirme sürecine daha teknik ve görev odaklı ilerlenmesi planlanmaktadır.
