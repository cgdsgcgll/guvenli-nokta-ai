# Backlog Dağıtma Mantığı

Sprint 1 sürecinde öncelik, ürünün teknik geliştirmesinden önce proje fikrini ve proje yönetimi yapısını netleştirmek olarak belirlenmiştir. Bu nedenle ilk sprintte product backlog, GitHub repository düzeni, sprint board, dokümantasyon ve ilk arayüz taslaklarına odaklanılmıştır.

## Sprint 1'e Alınan İşler

| İş | Neden Sprint 1'e Alındı? |
|---|---|
| GitHub repository oluşturma | Proje sürecinin tek bir merkezden takip edilmesi için gerekliydi. |
| README dosyasının hazırlanması | Ürün fikri, takım rolleri ve proje kapsamının görünür olması için gerekliydi. |
| Product backlog oluşturma | Ürünün geliştirilecek özelliklerini önceliklendirmek için gerekliydi. |
| Sprint board oluşturma | Görevlerin Todo, In Progress ve Done olarak takip edilmesi için gerekliydi. |
| Daily Scrum notları oluşturma | Takım içi ilerleyişin ve engellerin belgelenmesi için gerekliydi. |
| Wireframe tasarımlarını hazırlama | Ürünün temel ekranlarını görselleştirmek için gerekliydi. |

## Sprint 2'ye Bırakılan İşler

- Frontend arayüzlerinin React ile kodlanması
- Backend API yapısının kurulması
- Risk puanı algoritmasının geliştirilmesi
- AI öneri sistemi için ilk prototipin hazırlanması
- Kullanıcı form verilerinin işlenmesi
- Dashboard ekranının geliştirilmesi

## Sprint 3'e Bırakılan İşler

- AI chatbot geliştirmesi
- Test ve hata düzeltmeleri
- Kullanıcı deneyimi iyileştirmeleri
- Demo videosu hazırlanması
- Final teslim hazırlığı

## Değerlendirme

Sprint 1'de temel amaç, geliştirme sürecine başlamadan önce proje altyapısını ve ürün yönünü netleştirmekti. Bu nedenle kod geliştirme işleri sonraki sprintlere bırakılmıştır.
